# Variables for the SDF matrix prompt template (template.txt).
#
# Format:
#   {{varname}}  # optional description
#       plain value
#       0.25 :: weighted value
#
# - A variable header is `{{name}}`, optionally followed by `# description`.
# - Every non-empty indented line under a header is one value.
# - Full-line comments (lines whose first non-space char is `#`) are ignored
#   anywhere, so you can use `# --- section ---` dividers inside long value
#   lists.
# - Weights are optional: prefix a value with `<weight> :: `. Within one
#   variable they are all-or-nothing — either every value has a weight (and
#   they must sum to 1.0) or none does (= uniform). Weights are each value's
#   share of a sampled run; full enumeration ignores them.
# - Variables starting with an underscore (e.g. {{_preamble}}) are NOT matrix
#   axes — compose_prompts.py injects them (the preamble comes from
#   prompts/sdf/preamble.txt). Don't define them in this file.
#
# Running the composer (full options: python sdf_pipeline/compose_prompts.py --help):
#
#   # choose how many prompts this layer produces with --n-prompts:
#   python sdf_pipeline/compose_prompts.py --n-prompts 5000 --seed 0 --out prompts.jsonl
#
# --n-prompts deck-samples exactly n prompts whose per-variable distributions
# match the weights by construction (largest-remainder quotas, no sampling
# drift). Without --n-prompts you get the full cartesian product; without
# --out, a dry run that prints counts and three sample prompts.
