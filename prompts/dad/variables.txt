# Variables for the DAD scenario matrix (step1a_scenario.txt).
#
# Format (same as prompts/sdf/variables.txt):
#   {varname}  # optional description
#       plain value
#       0.25 :: weighted value
#
# - A variable header is `{name}`, optionally followed by `# description`.
# - Every non-empty indented line under a header is one value.
# - Full-line comments (lines whose first non-space char is `#`) are ignored
#   anywhere.
# - Weights are optional: prefix a value with `<weight> :: `. Within one
#   variable they are all-or-nothing — either every value has a weight (and
#   they must sum to 1.0) or none does (= uniform). Weights are each value's
#   share of a sampled run.
# - Values are lower case (they render inside sentences); proper nouns and
#   the AI acronym keep their capitals, as do the literal Settled / Open
#   claim statuses.
# - Placeholder syntax is the pipeline-standard `{name}` (Python str.format),
#   so literal { } braces must not appear in step1a_scenario.txt or in values
#   here.
# - NONE VALUES: each optional axis ({secondary_value_pair},
#   {cultural_setting}, {frontier_frame}) has exactly one "none" value —
#   stored as null on the scenario record but still rendered into the
#   template, so write it as the sentence the planner should read in that
#   case. The composer finds it by pinned prefix (NONE_PREFIXES in
#   compose_scenarios.py: "none", "no particular", "the ordinary"); reword
#   freely as long as the leading words survive — a broken match fails
#   loudly at deal time.
# - Reserved slots (injected by compose_scenarios.py, never defined here):
#   {taxa_hint} and {taxa_subcategory} (looked up from the drawn
#   {taxa_category} — the hint text and per-category species pools live in
#   TAXA at the top of compose_scenarios.py); {secondary_domain_clause} and
#   {secondary_goal_clause} (a 30% coin per scenario adds a second domain/goal
#   drawn from these same lists).
#
# Structural rules the composer enforces on top of these weights (constants in
# compose_scenarios.py, mirroring the old step1_dilemmas.py sampler). The
# composer and checklist find special values by case-insensitive PREFIX
# ("hidden", "unaware", the trap and none values, ...), so rewording a value
# is safe as long as its leading words survive; a broken match fails loudly
# at deal time, before any API spend:
#   - trap surface form -> hidden visibility -> unaware attitude (the only
#     sanctioned axis dependencies);
#   - no domain above 12% of a run, counting secondary domains;
#   - {length} values map to lenient acceptance bands (LENGTH_BANDS in
#     compose_scenarios.py, matched by leading-words prefix) that reject only
#     egregious 1b misses; {taxa_category} values map to their hint/species
#     entries (TAXA) the same way.
# There are deliberately NO small-run presence floors: the weights alone
# decide what a run contains, so a small smoke run may miss a rare slice
# (a trap, a frontier frame, a specific length register).
#
# Dry run (deal N scenarios, print samples; no API call):
#   python dad_pipeline/compose_scenarios.py --n 10 --seed 0
